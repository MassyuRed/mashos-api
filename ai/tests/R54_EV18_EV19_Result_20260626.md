## 3. 必要に伴う影響修正

EV19 で P8材料候補を正しく集約するため、EV12 / EV16 / EV17 に次の body-free count を伝搬させた。

```text

```

理由:

```text
primary_class_counts 全体をそのまま P8材料にすると、no_question_needed_emlis_can_observe まで P8材料として見えてしまう。
EV19 は question need observation rows のうち、plan_candidate_flags.p8_design_material_candidate=true かつ P8材料として許される primary class だけを集約する必要がある。
```

この変更は schema / body-free helper 内の集計伝搬であり、API / DB / RN / runtime / public response contract は変更していない。

## 8. 華恋の判断

EV18 / EV19 は、前に進めるための許可ではなく、次の判断材料を分けるための工程。

そのため今回は、P8材料候補を集約しつつ、question text / question implementation / P8 start はすべて false のまま固定した。

次の EV20 では、今回作った P6 / P8 candidate-only handoff も含めて、body-full / question text / no-touch violation が成果物に混ざっていないかを最終確認する必要がある。
