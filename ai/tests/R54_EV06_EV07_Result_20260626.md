## 3. EV06 固定内容

EV06では、body-full packet本体ではなく、後続のlocal-only packet生成へ渡すための body-free request evidence だけを固定しました。

残してよいもの:

```text
packet_generation_request_ref
packet_generation_request_policy_ref
packet_request_count
packet_ref_ids
allowed_output_ref = local_only_body_full_packet
forbidden_output_refs
export_denylist_refs
```

残さないもの:

```text
packet content
raw input
returned body
history surface
local path
body hash
question text
draft question text
reviewer free text
```

EV06は、EV05 manifest ready の場合だけ ready になります。EV05が blocked の場合は、packet generation request も blocked です。

## 9. 華恋メモ

EV06で一番危ないのは、body-full packetを生成していないのに、生成したかのように見える証跡を作ることです。  
今回は `request body-free化` に閉じ、packet本体・local path・body hashを残さないようにしました。

EV07も、実際のlocal生成手順を成果物へ書き出すのではなく、成果物に混ぜないための境界指示として固定しました。  
この分離を維持したままEV08以降へ進むのが安全です。
