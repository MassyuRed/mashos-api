## 7. 境界メモ

今回のtest rows / source materialsは body-free synthetic contract rows である。  
CLR-20 / CLR-21 の candidate-only handoff境界が成立することは確認したが、実際に人間が24件を読んだ証明ではない。

特に、次を混同しない。

```text
P6 candidate-only handoff != P6 start
P8 material candidate-only handoff != P8 start
P8 material candidate count > 0 != question implementation permission
helper green != actual human review complete
candidate != final
```

華恋の意見として、今回の実装では「候補を作ること」と「開始すること」を分けたことが大事だった。  
P6もP8も、ここでstartへ進めると、R54 actual review evidenceの線が短絡する。今回の段階では、Cocolonの記録線を守るため、候補だけをbody-freeで渡し、開始許可はfalseのまま保持する。
